Making Moodle Multi-device Friendly with Bootstrap

Dressing for success in a Multi-Device World, Twitter: @basbrands

Abstract: During the Ireland and UK Moodle Moot 2013 Hackfest the Bootstrap framework was discussed and the integration of a Moodle Bootstrap theme into Moodle core started. Since Moodle version 2.5 the Bootstrap framework is part of Moodle core and can be used for creating themes, plugins and content.

The first part of this presentation will show some examples of Bootstrap themes with different approaches to multi-device support.

The second part will show how the bootstrap framework can be used when creating plugins. The last part will show how teachers and course creators with some basic html knowledge can create interactive and attractive content using Bootstrap markup.

Bootswatches

Using bootswatches it's easy to style a theme. The Amelia and Cerulean themes have been ported from Bootstrap version 2. Basically a Bootswatch contains 2 files, one containing all colour, sizes and positioning (the variables) and one that overrides standard bootstrap styling (the bootwatch). When these are compiled they are not added to the theme, the override the theme.

Bootswatches

In the development process of Bootstrap3 it has proven to be very helpful to test with Bootswatches to see if the Moodle theme we have created complies with the Bootstrap way of using variables and Bootswatches.

I how from this presentation you have learned Bootstrap is a great framework for connecting developers with designers. If they both understand Bootstrap the results can be amazing. In Moodle you can allow your teachers to get the most out of a Bootstrap theme, just by some simple configuration in your TinyMCE editor settings.

Add the word styleselect to get access to the Tiny MCE styleselector.

Tiny MCE Labels
